# 문구 라이브러리 (phrase-library)

**패턴에 사실을 채워 쓰는 라이브러리다. 완성문 복붙 금지** — `{슬롯}`을 실제 사실로
채우지 못하면 그 문장은 쓰지 않는다(빈 슬롯 채우려 사실을 지어내는 것이 최악의 실패).

## 사용 규칙

1. `{}`는 사실 슬롯: RFP·고객 자료·검증된 웹 정보로만 채운다.
2. 슬롯을 채울 근거가 없으면 → `evidence-and-claims.md`의 `[unverified]` 처리 절차.
3. 같은 패턴을 한 문서에서 3회 이상 반복하면 표현을 변형한다(복붙 티 방지).

## A. 목적·배경 (Ⅰ장)

| 용도 | 패턴 |
|---|---|
| 사업 배경(3단 논리) | `{고객}은(는) {환경 변화/현황} 상황에서 {당면 과제}에 직면해 있으며, 본 사업은 {목적}을 달성하고자 합니다.` |
| 목적 요약 | `본 제안은 {핵심 요구 1~3개 요약}을 위한 {솔루션/서비스}의 {구축/제공} 방안을 제시합니다.` |
| 추진 방향 | `{고객 환경 이해}를 바탕으로 {고려요소}를 반영한 {N}대 추진전략을 수립하였습니다.` |

## B. 역량·실적 (Ⅱ장)

| 용도 | 패턴 |
|---|---|
| 회사 소개 | `당사는 {설립연도}년 설립 이래 {전문 분야}에 전념해 온 {포지셔닝}이며, {대표 근거: 파트너 등급/고객 수/유지율}을 보유하고 있습니다.` |
| 실적 강조 | `{산업군} {고객 수}개사 대상 {서비스} 수행 실적({기간}, 별첨 실적표 참조)을 통해 {발주처} 환경에 검증된 방법론을 적용합니다.` |
| 인력 역량 | `투입 인력 {N}명 전원 {자격/등급} 보유, 평균 경력 {N}년({기준일} 기준)입니다.` |
| 파트너십 | `{벤더} {파트너 등급} 자격으로 {혜택: 기술지원 에스컬레이션/로드맵 사전 공유/교육}을 제공받아 지원합니다.` |

## C. 기술 방안 (Ⅲ장)

| 용도 | 패턴 |
|---|---|
| 아키텍처 결론 | `{구성 방식}으로 {품질 속성: 가용성/확장성/보안성}을 확보합니다.` 예: "관리 노드 3중화와 로드밸런서 이중화로 단일 장애점 없는 관리 체계를 구성합니다." |
| 제품 선정 사유 | `{제품}은 {객관 지표: 시장 점유율/커뮤니티 규모/지원 기간}에서 {비교 우위}를 보이며(출처: {기관, 연도}), {고객 요구 X}에 부합합니다.` |
| 단계별 로드맵 | `1단계({기간}) {범위} → 2단계({기간}) {범위} → 3단계({기간}) {범위}로 위험을 분산하며 확장합니다.` |
| 마이그레이션 | `{현행 환경}에서 {목표 환경}으로 {방법론/도구}를 활용해 이관하며, {검증 절차}로 전환 위험을 통제합니다.` |
| 미지원 항목(유형 C) | `현재 {기능}은 미지원이나, {대안} 적용이 가능하며 {버전/시점} 로드맵에 반영되어 있습니다.` (공란·회피 금지) |

## D. 지원·SLA (유지보수형)

| 용도 | 패턴 |
|---|---|
| 지원 체계 | `1선({전담 조직}) → 2선({본사 기술팀}) → 3선({제조사 글로벌 기술지원 조직})의 3단계 에스컬레이션 체계로 {목표: 신속한 원인 분석과 해결}을 보장합니다.` |
| SLA 약속 | `심각도 {등급} 장애 시 {응답 시간} 이내 응답, {복구 목표}를 목표로 대응합니다(세부 기준: SLA 표 참조).` |
| 예방 활동 | `{주기} 정기점검({N}개 항목)과 {산출물: 점검확인서/월간보고서} 제출로 장애를 사전 예방합니다.` |
| 교육 제공 | `운영자 대상 {과정명} 교육({시간/회수})을 무상 제공하며, {벤더 공식 교육}은 {조건}으로 지원합니다.` |

## E. 일정·관리 (Ⅳ장)

| 용도 | 패턴 |
|---|---|
| 일정 약속 | `총 {기간} 중 {마일스톤}을 {시점}까지 완료합니다(전제: {가정 — 장비 납기/고객 승인 일정 등}).` |
| 위험 관리 | `{위험 항목}에 대해 {완화 방안}을 사전 적용하고, 발생 시 {대응 절차}로 영향도를 {목표 수준} 이내로 관리합니다.` |
| 품질 보증 | `{방법론/표준}에 따라 {검토 단계}를 수행하며, 산출물은 {검수 절차}를 거쳐 제출합니다.` |

## F. 기대효과 (정량 주장)

| 용도 | 패턴 |
|---|---|
| 정량 효과 | `{항목} {수치}{단위} {개선/절감} 예상(산식: {산식}, 전제: {전제조건}).` 산식·전제 없이 수치만 쓰지 않는다. |
| 정성 효과 | `{활동}을 통해 {효과: 운영 표준화/기술 내재화/대응 속도 향상}을 기대할 수 있습니다.` |

## 신규 문구 등록 (라이브러리 확장)

반복 사용할 문구·규칙을 추가할 때는 아래 스키마로 이 파일 하단에 누적한다:

```yaml
- id: PHR-001
  category: 실적강조            # 목적·배경/역량·실적/기술방안/지원SLA/일정관리/기대효과
  rule: "패턴 문장 ({슬롯} 포함)"
  applies_to: [A, B, C]         # 적용 제안 유형
  priority: high                # high/medium/low
  good_example: "실제 채워 쓴 좋은 예"
  bad_example: "슬롯 없이 과장한 나쁜 예"
  evidence_required: true       # 근거 표·출처 병기 필요 여부
  source: "출처 제안서/가이드"
  owner: "등록자"
  reviewed_at: "YYYY-MM-DD"
```
